Official Recruit-Recruiters' scraper

Below you can find all the endpoints available (Note that this scraper is tightly coupled with the recruit-recruiters frontend, thus making some requests confusing)

  • "/jobs" uses a GET request, which takes up to three query parameters named title, location and page.

    title stands for the job title

    location stands for the job location

    page stands for the page to scrape data on from every job hosting site that the app can scrape

    Example: "/jobs?title=Java&location=Cluj-Napoca&page=2" will return all the Java related offers from Cluj-Napoca from page 2

    Any of these params can be removed!

  • "/job/details" uses a PUT request, which needs a body under the form of a JSON object like:

    {
    "applicationLink": string,
    "companyImage": string,
    "companyName": string,
    "description": string,
    "jobTitle": string,
    "location": string,
    "offerLink": string,
    "site": string,
    "workType": string
    }

    The main key being offerLink. That is from where the details of the job like company Image or work type will be scraped form.
